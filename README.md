# 🎙 Интервью Ассистент

Локальный инструмент для записи и транскрибации интервью.  
**Все данные остаются на вашем компьютере. Никакого облака.**

---

## Быстрый старт

1. Разархивируйте папку `InterviewAssistant`
2. **Дважды щёлкните** `Start Interview Assistant.command`
3. При первом запуске зависимости установятся автоматически (~3–5 мин)
4. Браузер откроется сам → приложение готово

> **Если файл не открывается:** щёлкните правой кнопкой → «Открыть»,
> или один раз в Терминале: `chmod +x "Start Interview Assistant.command"`

---

## Использование

| Шаг | Действие |
|-----|----------|
| 1 | Введите имя кандидата |
| 2 | Нажмите **Начать запись** — разрешите доступ к микрофону |
| 3 | Проведите интервью |
| 4 | Нажмите **Остановить запись** |
| 5 | Дождитесь обработки (несколько минут для длинных записей) |
| 6 | Читайте транскрипт, резюме и заметки прямо в браузере |
| 7 | Скачайте нужные файлы кнопками **⬇ .md** |

## Что создаётся для каждого интервью

```
outputs/
└── 2026-05-14_ivanov-ivan_a3f2c8b1/
    ├── recording.webm           — оригинальная запись
    ├── transcript_raw.txt       — сырой текст без меток
    ├── transcript_cleaned.txt   — текст с именами спикеров
    ├── transcript_diarised.md   — транскрипт с временными метками
    ├── summary.md               — структурированное резюме
    ├── recruiter_notes.md       — шаблон заметок рекрутера
    └── metadata.json            — метаданные сессии
```

## Настройка модели Whisper

По умолчанию используется модель `medium` (рекомендуется).  
Чтобы изменить, откройте `whisper_worker.py` и найдите строку:

```python
MODEL_SIZE = "medium"
```

Доступные варианты:

| Модель | Размер | Скорость | Точность |
|--------|--------|----------|----------|
| `tiny` | 75 MB | ⚡⚡⚡ | ★★☆ |
| `base` | 145 MB | ⚡⚡ | ★★★ |
| `small` | 484 MB | ⚡ | ★★★★ |
| `medium` | 1.5 GB | ◎ | ★★★★★ (рекомендуется) |
| `large-v3` | 3 GB | slow | ★★★★★+ |

## Диаризация (разделение по спикерам)

Используется простой алгоритм: длинные паузы между репликами считаются
сменой спикера. Первый спикер — **Интервьюер**, второй — **Кандидат**.

Это работает хорошо для структурированных 1-на-1 интервью.  
Для более точной диаризации можно добавить `pyannote.audio` (требует
HuggingFace-токен и ~1-2 GB дополнительных моделей) — обратитесь к команде.

## Конфиденциальность

- Аудио не покидает компьютер ни при каких условиях
- Транскрибация выполняется локально через Whisper
- Нет телеметрии, нет аналитики, нет сетевых запросов после первой установки
- Все файлы в папке `outputs/` — только ваши

## Отладка

**Сервер не запускается:**
```bash
# Запустите вручную для просмотра ошибок
cd /путь/до/InterviewAssistant
.venv/bin/python app.py
```

**Микрофон недоступен:**  
Системные настройки → Конфиденциальность и безопасность → Микрофон →
включите доступ для браузера (Chrome/Safari/Firefox).

**Порт 8000 занят:**  
```bash
lsof -i :8000   # узнать что использует порт
```

**Логи сервера:**  
Файл `logs/server.log` содержит все сообщения сервера.

## Ручной запуск (альтернатива лаунчеру)

```bash
cd /путь/до/InterviewAssistant
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
.venv/bin/python app.py
# → открыть http://localhost:8000
```
